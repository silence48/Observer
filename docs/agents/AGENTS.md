# Specialist Agent Index

Read `../../AGENTS.md` first. These files define named specialists for work that
crosses package boundaries. Use them as role prompts when a task touches more
than one folder or needs a specific lens.

- `vera-the-archive-sentinel.md` - archive verification and rescans.
- `orla-the-orchestrator.md` - subagent coordination and roadmap control.
- `max-the-memory-mechanic.md` - worker pools, parallelism, heap safety.
- `nova-the-scp-cartographer.md` - SCP telemetry and visualization.
- `bill-the-graph-theory-genius.md` - quorum, FBAS, graph theory, graph stores.
- `maya-the-search-smith.md` - Meilisearch and fast read models.
- `heidi-the-horizon-keeper.md` - full-history ETL, Horizon, Soroban RPC.
- `alvin-the-network-god.md` - live network scanner, validators, TOML, quorum.
- `steve-the-typescript-junkie.md` - TypeScript 6 and strict refactors.
- `bella-the-reverse-engineering-guru.md` - RADAR parity and external audits.
- `jim-the-docs-guy.md` - docs, API docs, roadmap alignment.
- `silence-the-1337.md` - operations, BrowserOS, deploy/runtime safety.
- `quinn-the-quality-bouncer.md` - QA, build logs, release hygiene.

All specialists follow the same base rules: files under 500 lines, strict
TypeScript, short efficient implementations, bounded concurrency, and no
duplicate persisted archive data.
